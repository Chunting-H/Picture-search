# AGENTS.md

## 项目背景

本目录用于开发本地图片搜索产品的原生 macOS App。

项目目标是开发一个通过 Apple 官方 API 读取用户 Photos 图库、在本机建立 OCR 和图片语义索引，并支持用户用自然语言找图的 macOS App。

## 不可违背的产品规则

1. 只使用 PhotoKit 读取 Photos 资产，不直接访问、检查、解析或依赖磁盘上的 Photos Library 文件包。
2. App 默认只读。除非未来需求明确允许，否则不得修改、删除、移动、标记、整理或以其他方式改变用户照片。
3. OCR、embedding、向量搜索、元数据提取和索引默认在本机运行。不得把用户照片、OCR 文本、embedding、索引或派生的私密元数据上传到网络服务。
4. 不实现人物识别、人脸身份识别、人脸聚类或身份推断。
5. 每次只完成一个明确里程碑。实现里程碑时不得进行大范围重写、整体重构或无关清理。
6. 修改前先阅读 PRD、`PLAN.md` 和 `docs/决策记录.md`。如果其中某些文件不存在，应在工作总结中说明，不得虚构其内容。
7. 新增外部依赖前，必须说明依赖原因、考虑过的替代方案，以及对隐私、App 体积、构建复杂度、可维护性和许可证的影响。
8. 每次修改后，运行当前环境能够执行的构建或测试。
9. 构建失败时先定位根因，不得通过删除需求、移除功能或弱化预期行为来绕过问题。
10. 每完成一个里程碑，同步更新 `docs/开发记录.md`、`docs/已知问题.md` 和 `README.md`。
11. 不把尚未验证的能力写成已经完成。应按实际情况标记为计划中、部分完成、实验性或未验证。
12. 所有用户可见错误都要有明确说明和具体恢复动作。
13. 所有解释、`README.md`、`docs/` 下文档、开发记录、决策记录、验收记录和面向用户或协作者的说明性内容都要用中文编写。必要的技术名词、API 名称、命令、文件名和代码标识符可以保留英文。

## 开发工作流

- App 实现工作放在 `code/` 目录内。
- 修改范围应限定在当前里程碑及其必要文件内。
- 在能满足需求时，优先使用原生 macOS 和 Apple 框架能力。
- 在默认配置、日志、测试、测试数据和文档中都保持隐私优先。
- 避免记录照片内容、OCR 文本、embedding、暴露私有图库结构的文件路径或其他敏感派生数据。
- 异步任务和索引流程应尽量支持恢复。索引应能容忍中断、App 重启、重复发现资产和单个资产读取失败。
- iCloud 资产是正常场景。App 必须用清晰状态和恢复建议处理不可用或尚未下载的资产。
- 保持 UI 响应流畅。不得在主线程执行 Photos 拉取、OCR、embedding 生成、图片解码或索引写入。

## 修改前必须阅读

编辑实现文件前，先检查当前产品和计划文档：

- PRD：优先阅读 `../04_prd/本地图片智能搜索_PRD_v1.md`，除非后续引入了更新版本。
- 计划：阅读 `PLAN.md`；如果不存在，则阅读最接近当前工作的计划文档。
- 决策：阅读 `docs/决策记录.md`，如果该文件存在。

如果这些文件缺失或内容不完整，应基于现有上下文继续，并在最终总结中记录信息缺口。

## 构建、测试与验证

- 每次修改后，运行当前环境可用的构建或测试命令。
- 如果尚无构建系统，应尽可能直接验证改动文件，并说明当前没有可执行的构建或测试目标。
- 对 PhotoKit、权限、索引、OCR 和搜索行为，测试应覆盖失败路径和正常路径。
- 在实现、验证和必要文档更新完成前，不得把里程碑标记为完成。

## 文档要求

所有解释、`README.md`、`docs/` 下文档、开发记录、决策记录、验收记录和面向用户或协作者的说明性内容都要用中文编写。必要的技术名词、API 名称、命令、文件名和代码标识符可以保留英文。

每个里程碑结束时，更新：

- `docs/开发记录.md`：记录改了什么、验证了什么、还剩什么。
- `docs/已知问题.md`：记录当前限制、风险和未解决失败。
- `README.md`：准确记录设置、构建、使用方式和能力状态。

文档必须区分已完成能力、计划中能力和未验证能力。

## 代码审查重点

审查者和代理应特别关注：

- PhotoKit 权限处理、隐私边界和只读行为。
- 异步任务、取消、内存占用、背压和主线程阻塞。
- 索引状态丢失、重复处理、可恢复性和并发写入。
- 图片读取失败、降级资产和未下载的 iCloud 资产。
- 测试和文档是否随代码同步更新。
